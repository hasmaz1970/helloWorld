    MonE L1 Architecture

    The goal of L1 Architecture section is to provide how the System Domains that constitute MonE Platform work together in harmony. The following subsections provide further information considering the following perspectives:

    • The Platform Functional Domains (Public API, Charging and Billing) that denote the main building blocks which constitute the MonE Platform and interact via Reference Points
    • The Data Access and Exchange Domain that denote the data access services for data at rest and data in motion
    • The Operational Domain that denote additional container platform orchestration, operational intelligence and API Management services.
    • Reference Points that denote a systematic grouping/classification of related API(s) between any two domains/components of MonE Platform. Thus, reference points form the critical interconnection contracts among two entities within the architecture.


    The following diagram depicts MonE Level 1 Architecture.

    Figure-1: MonE L1 Architecture

    1. Platform Functional Domains

    1.1. MonE Public APIs Domain

    Public APIs domain programmatic interface comprises all the APIs for interactions between MonE and External Client for a particular purpose that wants to access the externally accessible MonE capabilities, such as: Charging Management, Account Management, Order Management, Payment Management etc. services.

    The components in this domain access and update information on MonE_DB and also publish/subscribe to relevant topics on Event/Message Streaming Platform that allow triggering cache updates, charging and billing back-end tasks.

    1.2. MonE Charging Domain

    MonE Charging Domain comprises the components that provide the end-to-end functionality for the entire charging chain whether be offline and/or online charging including simple or complex/cross discounts that depend on aggregated usage via a balance manager.

    The components in this domain access and update information via MonE In-Memory Cache and also publish/subscribe to relevant topics on Event/Message Streaming Platform that allow reliable and low-latency exchange of information among charging components. Charging Components have no database dependency except for the final stage of charging where charged data transactions and balance information is persisted on MonE_DB.

    1.3. MonE Billing Domain

    MonE Billing Domain comprises the components that provide the end-to-end functionality of the entire billing chain whether be for hot-billing, on-demand-billing or batch-bill generation purposes including bill time discounts that that depend on aggregated bill line item amounts via a balance manager.

    The components in this domain access and update information via MonE In-Memory Cache and also publish/subscribe to relevant topics on Event/Message Streaming Platform that allow exchange of information among billing components. Billing Components have additional database dependency such as to persist billing results, adjust bills, register payments etc. on the MonE_DB.

    2. Data Access and Exchange Domain

    MonE Data Access and Exchange Domain comprises the 3PP components that provide the following end-to-end capabilities for the entire MonE Platform:

    • MonE_DB (RDBMS) provides persistent storage, update and access services for data at rest whether be reference configuration data or transactional data.
    • MonE_InMem_Cache provides in-memory low-latency access to data for high-performance computing needs of charging and billing modules.
    • Event/Message Streaming Platform (EMSP) provides distributed, highly-scalable, durable and high-ingress event data streams broker services for inter-component integration among back-end platform components.
    • MonE_Filesystem provides persistent disk volume space to hold logs or use as scratch area by components.

    3. Operational Domain

    MonE Operational Domain comprises the 3PP components that provide the following end-to-end capabilities for the entire MonE Platform:

    • Container Orchestration and Management Platform provides deployment, scaling and other component lifecycle management services for the containerized MonE components.
    • Intelligence and Analytics Management with all MonE components providing out-of-the-box transactional and kpi data feed capabilities, this component enables continuous end-to-end monitoring of the distributed platform with full-blown operational analytics capabilities.
    • API Gateway (Not provided with MonE) In real-life scenarios, as-is BSS/OSS ecosystem of enterprise organizations will mandate integration to one or more legacy systems over proprietary APIs, the API Gateway component enables such custom integration services to MonE.
