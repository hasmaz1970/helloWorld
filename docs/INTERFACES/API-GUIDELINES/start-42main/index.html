    Overview

    MonE Public APIs and Data Models follow TMFORUM open API recommendations as detailed in (Ref. [1]). This section is intended for anyone wanting to understand the overall protocol aspects of REST/HTTP API(s) provided by MonE Components that expose functionality of each self-contained system building block to external clients as well as other MonE components working in harmony.

    For detailed specification of each API endpoint and schema definitions of object types, please refer to MonE Public APIs Specification.

    1. General

    The following table captures the very high-level important aspects of MonE APIs in a nutshell.

    PROPERTY VALUE DESCRIPTION
    TRANSPORT LAYER PROTOCOL TCP Referring to Internet Protocol Suite, Transport Layer is Transmission Control Protocol that is over IP in the protocol suite.
    APPLICATION LAYER PROTOCOL HTTP/1.1 Referring to Internet Protocol Suite, Application Layer is Hypertext Transfer Protocol (HTTP) ver 1.1. (RFC 2616), (RFC 7230)
    API STYLE REST See next 2 rows
    ALLOWED HTTP METHODs GET,
    POST,
    PUT,
    PATCH,
    DELETE     		GET for query/read purposes. POST/PUT/PATCH/DELETE for update and/or invocation of a task purposes
    CONTENT TYPE JSON All exchanged submitted and response payload is encoded as JSON
    CHARACTER SET/ENCODING Unicode/ UTF-8 Information exchanged compliant to JSON style content-type serialization is encoded in UTF-8
    PROTOCOL SECURITY
    AUTHENTICATION    		 HTTPS Hypertext Transfer Protocol Secure (HTTPS). In HTTPS, the HTTP/1.1 protocol is encrypted using TLS (Transport Layer Security) cryptographic protocol (or formerly SSL-Secure Sockets Layer). HTTPS provides: a)authentication of the accessed website, and b)protection of the privacy and integrity of the exchanged data while in transit, by encrypting communication between client and server.
    PROTOCOL SECURITY
    AUTHORIZATION    		 OAuth 2.0 OAuth separates the authorization provider from the resource. The authorization server issues access tokens which are used to access the protected resource. This allows consumption of the API without sharing credentials with the API provider.
