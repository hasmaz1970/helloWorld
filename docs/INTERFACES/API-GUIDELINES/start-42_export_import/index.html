    Export, Import Patterns

    Task Resource Patterns already details the Task Resource patterns. This section provides further details specific to two critical common tasks:

    • Import Job(s): used to import resources from a File
    • Export Job(s): used to export resources to a File

    1. Import Jobs and Operations

    The following table depicts the Import Job resource model.

    Field Description

    id

    string

    Identifier of the job

    href

    string

    Reference of the job

    completionDate

    ($date-time)

    Date at which the job was completed

    contentType

    string

    Indicates the format of the imported data

    creationDate

    ($date-time)

    Date at which the job was created

    errorLog

    string

    Reason for failure if status is failed

    path

    string

    URL of the root resource where the content of the file specified by the import job must be applied

    url

    string

    URL of the file containing the data to be imported

    Mandatory while creating an import job

    status

    string

    Valid values for the state of a batch job (e.g. catalog import)

    Enum:

    Array [ 4 ]

    @baseType

    string

    When sub-classing, this defines the super-class

    @schemaLocation

    ($uri)

    A URI to a JSON-Schema file that defines additional attributes and relationships

    @type

    string

    When sub-classing, this defines the sub-class entity name


    The following sub sections provides examples for Import Operations.

    1.1. Create Import Job

    Import Job Tasks are created as resources. The Import Job must be attached to a specific resource acting as the root for the collection of resources to be streamed to a File.

    As an example, an Import Job can be attached to a specific Catalog in a Catalog application or may be attached to the Product Offerings within a Catalog.

    • {apiRoot}/catalogManagement/catalog/42/importJob: Import all the resources within the Catalog 42
    • {apiRoot}/catalogManagement/catalog/42/productOffering/importJob: Import all the ProductOffering resources

    Example

    Create Import Job

    REQUEST 

    POST catalogManagement/catalog/{10}/importJob 
Content-type: application/json

{
   “path”: “catalogManagement/catalog/10" ,   
   "url": "ftp://ftp.myCatalog.com/productCatalog/partner10",
}


    RESPONSE 

    201 Created
Content-Type: application/json

{
  "id": "554",
  "href": "/catalogManagement/importJob/554", 
  "status": "running",
  “path”: “catalogManagement/catalog/10" , 
  “content-type”: “application/json”,
  "errorLog": "",
  “creationDate” : “2020-04-19T16:42:23-04:00",
  “completionDate” : “",
  "url": "ftp://ftp.myCatalog.com/productCatalog/partner54"
}

    1.2. Query Import Job

    Example

    Query Import Job and Retrieve its Status

    REQUEST 

    GET /catalogManagement/importJob/554
Accept: application/json


    RESPONSE 

    200 OK
Content-Type: application/json

{
  "id": "554",
  "href": "/catalogManagement/importJob/554", 
  "status": "completed",
  “path”: “catalogManagement/catalog/10" , 
  “content-type”: “application/json”,
  "errorLog": "http://my-platform/daily/logging/errors.log",
  “creationDate” : “2020-04-19T16:42:23-04:00",
  “completionDate” : “2020-04-19T16:42:23-18:00",
  "url": "ftp://ftp.myCatalog.com/productCatalog/partner54"
}

    1.3. Delete Import Job

    Example

    Delete Import Job

    REQUEST 

    DELETE /catalogManagement/importJob/554


    RESPONSE 

    204 No Content

    1.4. Import Job Completion Notification

    Please refer to Event, Hub and Notification Management Patterns section for details on Notification Patterns. Compliant to Notification patterns supported by MonE APIs, provided that a relevant callback API is registered via a hub as detailed in relevant section, a notification callback will be posted to the registered listener endpoint.

    Example

    Import Job Completion Notification

    POST /client/listener 
Content-Type: application/json

{
  "@type": "event",
  "@schemaLocation": "https://bss.operator.com/MonE/schemas/Event.schema.json",
  "@baseType": "event",   
  "eventId": "256c42f0-7cae-4cfe-8b96-f5773796f8ff",
  "eventTime": "2020-04-19T16:42:23-18:00",
  "eventType": "importJobCompleted",
  "priority": "Normal",
  "event":  {
    "id": "554",
    "href": "/catalogManagement/importJob/554", 
    "status": "completed",
    “path”: “catalogManagement/catalog/10" , 
    “content-type”: “application/json”,
    "errorLog": "http://my-platform/daily/logging/errors.log",
    “creationDate” : “2020-04-19T16:42:23-04:00",
    “completionDate” : “2020-04-19T16:42:23-18:00",
    "url": "ftp://ftp.myCatalog.com/productCatalog/partner54"
  }
}

    2. Export Jobs and Operations

    The following table depicts the Export Job resource model.

    Field Description

    id

    string

    Identifier of the job

    href

    string

    Reference of the job

    completionDate

    ($date-time)

    Date at which the job was completed

    contentType

    string

    Indicates the format of the exported data

    creationDate

    ($date-time)

    Date at which the job was created

    errorLog

    string

    Reason for failure if status is failed

    path

    string

    URL of the root resource where the content of the file specified by the export job must be applied

    query

    string

    Optionally used to scope the exported data

    url

    string

    URL of the file containing the data to be exported

    If NOT provided while creating an export job, the the server provides the URL for the output exported context in response to create request

    status

    string

    Valid values for the state of a batch job (e.g. catalog export)

    Array [ 4 ]

    @baseType

    string

    When sub-classing, this defines the super-class

    string

    When sub-classing, this defines the sub-class entity name
