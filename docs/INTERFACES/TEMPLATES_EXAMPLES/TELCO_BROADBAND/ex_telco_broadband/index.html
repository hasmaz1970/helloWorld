    Charging: Sample CDR

    2020-12-25 12:52:49;23;66.111.111.111;;77.122.122.222;;2163421559;12164558080;02164558080;OV;01;N;123456789123456789;;;
02163421559;9;123456789;2020-12-25 12:53:13;111@originating.domain.com;222@terminating.domain.com;
GW_DISTRICT_ABC_1.BT;HOST_C_ONNET_SIP_ABC_1.BT;02163421559;;

    The following is a description of each field [no], with index starting from 0:

    • [00]: Local GW start date and time: 2018-12-25 12:52:49
    • [01]: Duration: 23
    • [02]: Originator IP: 66.111.111.111
    • [03]: Originator Line #:
    • [04]: Terminator IP: 77.122.122.222
    • [05]: Terminator Line #:
    • [06]: User ID: 2163421559
    • [07]: Normalized Called Number (E.164): 12164558080
    • [08]: Dialed Number (E.164): 02164558080
    • [09]: Service Type Descriptor: OV
      • 1st character indicates direction: I:IN, O:Out
      • 2nd character indicates service type: V:Voice
    • [10]: Call Participants Descriptor: 01
      • 01: Phone to phone
      • 02: Desktop to phone
      • 03: Phone to Desktop
      • 10: Phone to PBX
      • 11: Desktop to PBX
      • 20: Web to PBX
    • [11]: Call Info Disconnect Reason: N
      • N: Normal
      • D: Disconnected
      • A: Abandoned
      • B: Busy
      • E: Error
    • [12]: Unique Call ID: 123456789123456789
    • [13]: Error Code (if any):
    • [14]: Error Description (if any):
    • [15]: ANI: Automatic Number Identification (Originating): 02163421559
    • [16]: DNIS: Dialed Number Identification Service GW/RAS Access Number: 9
    • [17]: CDR Sequence Number: 123456789
    • [18]: Local GW stop date and time: 2020-12-25 12:53:13
    • [19]: Originating Domain ID: 111@originating.domain.com
    • [20]: Terminating Domain ID: 222@terminating.domain.com
    • [21]: Originating Trunk ID: GW_DISTRICT_ABC_1.BT
    • [22]: Terminating Trunk ID: HOST_C_ONNET_SIP_ABC_1.BT
    • [23]: ANI Info Digits: 02163421559
    • [24]: Extra Info Digits:

    Charging: Sample Usage Tariff RuleSet

    The following illustrates a sample charging tariff that allows applying different rates to calls destined to USA and outside of USA.

    The payload in ruleset definitions below refers to CDS (Charging Data Stream) data type.

    • For calls destined to USA TieredRateFunction is applied.
    • For calls destined outside of USA, different LinearRateFunction(s) as per tariff rules considering timeOfDay for the calls.


    For reader's convenience, in the following, the request body is provided as a YAML representation. 

    POST https://monetizehub.io/demos/current/apis/public/admin/charging/tariffRuleSet
Content-Type: application/json


    '@schemaLocation': string
'@baseType': string
'@type': string0
name: 'VoIP_Tariff1234'
externalId: 'VoIP_Tariff1234'
version: '1.0'
validFor:
  '@schemaLocation': string
  '@baseType': string
  '@type': string
  name: string
  startDateTime: '2020-10-01T09:13:16-07:00'
rule:
  - name: rule_checkIF_USA
    branchName: InsideUSA
    type: IF
    condition:
      - 'payload.qualifierMap.get("NORMALIZED_DIALED_NUMBER","ADDRESS",0).value.startsWith("\+?1")' 
    result:
      resultIndex: 0
  - name: rule_checkIF_OUTSIDE
    type: IF
    condition:
      - '! ( payload.qualifierMap.get("NORMALIZED_DIALED_NUMBER","ADDRESS",0).value.startsWith("\+?1") )' 
    result:
      rule:
        - name: rule_checkIF_OUTSIDE_and_weekday
          branchName: OutsideUSA_WeekDay
          type: IF
          condition:
            - 'ZonedDateTime.parse(payload.qualifierMap.get("NORMALIZED_DIALED","NUMBER_ADDRESS",0).value).getDayOfWeek().getValue() <= 5'
          result:
            resultIndex: 1
        - name: rule_checkIF_weekend
          branchName: OutsideUSA_WeekEnd
          type: IF
          condition:
            - 'ZonedDateTime.parse(payload.qualifierMap.get("NORMALIZED_DIALED","NUMBER_ADDRESS",0).value).getDayOfWeek().getValue() > 5'
          result:
            resultIndex: 2

result:
  - name: 'USA Calls Pricing Method'
    attribute:
      name: 'TieredRateFunction'
      attribute:
        - name: 'Tier-1: 0-120sec'
          attribute:
            - name: startValue
              value: '0'
            - name: endValue
              value: '120'
            - name: m
              value: '0.05'
            - name: c
              value: '60'
            - name: round
              value: 'up'
        - name: 'Tier-x: 120-...sec'
          attribute:
            - name: startValue
              value: '120'
            - name: m
              value: '0.10'
            - name: c
              value: '60'
            - name: round
              value: 'up'
  - name: 'non-USA WeekDay Calls Pricing Method'
    attribute:
      name: 'LinearRateFunction'
      attribute:
          attribute:
            - name: m
              value: '0.50'
            - name: c
              value: '60'
            - name: round
              value: 'up'
  - name: 'non-USA WeekEnd Calls Pricing Method'
    attribute:
      name: 'LinearRateFunction'
      attribute:
          attribute:
            - name: m
              value: '1.25'
            - name: c
              value: '60'
            - name: round
              value: 'up'
